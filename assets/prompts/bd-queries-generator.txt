### ANALYSIS PROCESS

1. **IDENTIFY THE CORE PROBLEM:**
   - Read the user context and question carefully
   - Determine the ACTUAL problem by understanding the situation, not just keywords
   - Common problem types include (but are NOT limited to):
     * Survival: escape, low HP, critical situation, being focused
     * Offense: need damage, need to kill, need to push, need to farm
     * Defense: need protection, need to survive, need to counter enemy strategy
     * Utility: need vision, need detection, need mana, need mobility
     * Strategy: timing decisions, positioning, team coordination, objective control
     * Counter-play: counter specific hero, counter specific item, counter specific strategy
   - If the problem doesn't fit any category, identify the core game mechanic involved

2. **MAP PROBLEM TO MECHANICAL TERMS:**
   - Match the problem to Dota 2 mechanical terminology (see terminology section below)
   - If problem is NOT in the terminology list, use general Dota 2 mechanics:
     * Need damage → "damage", "DPS", "physical damage", "magical damage"
     * Need farm → "farming", "gold", "farm acceleration", "wave clear"
     * Need mobility → "mobility", "movement speed", "blink", "teleport"
     * Need initiation → "initiation", "disable", "stun", "catch"
     * Need save → "save", "save item", "defensive", "protection"
     * Need utility → identify the utility type (vision, detection, mana, etc.)
   - Always use Dota 2 mechanical terms, NOT casual language
   - Examples:
     * Problem: "escape when low HP" → use "escape", "survival", "TP scroll", NOT "healing reduction"
     * Problem: "clear illusions" → use "illusion clear", "AoE damage", "area damage", NOT "healing reduction"
     * Problem: "survive control" → use "status resistance", "BKB", "dispel", NOT "damage reduction"
     * Problem: "need to farm faster" → use "farming", "farm acceleration", "wave clear", NOT "get more gold"

3. **GENERATE DIVERSE QUERIES:**
   - Cover different aspects: mechanics, items, hero counters, strategies
   - Each query must be RELEVANT to the actual problem
   - Do NOT force queries that don't match the problem (e.g., don't generate "healing reduction" for escape problems)

### HANDLING VAGUE QUESTIONS

If the question lacks specifics (e.g., "what item should I buy?"):

3. If no context → generate broad queries covering common needs:

4. AVOID guessing specifics without evidence
